Utiliser Git et Github pour vos travaux personnels et en équipe

  1. Traveiller de manière basique
    1. Vérifier que Git est installé git --version
    2. Définir l'utilisateur global git config --global user.name "Nom complet"
      git config --global user.email "monemail@exemple.com"
    3. Créer un nouveau project mkdir "nom du projet"
    4. Entrer et initialiser le dépôt cd "nom du projet"
      git init
    5. Valider les modifications éffectuées git status
      git add . (ou * ou encore nom-du-fichier.extension)
      git commit -m "Message pour le log et l'historique"       Ce processus est à faire à chaque modification du code source.
    6. Pour voir l'historique des modifications de botre code source git log
  2. Travailler avec les branches
    1. Créer une branche avec Git git branch "nom de la branche" Le nom de la branche doit avoir un lien avec la fonctionnalité traitée pour une meilleure organisation. Pour voir la liste des branches existantes git branch
    2. Pour changer de branche git checkout "nom de la branche" Les premières instructions sont valables sur n'importe quelle branche. Il faut donc faire attention à la branche sur laquelle nous travaillons.
    3. Finaliser les travaux sur la branche principale git checkout "master | main"
      git merge "nom de la branche à fusionner"
      git log
    4. Pour supprimer une branche git branch -d ",om de la branche à supprimer"
  3. Et si on travaillait en équipe
      Il faut faire un dépôt distant pour permettre aux autres de pouvoir travailler. Nous allons le faire avec Github.
    1. Créer un un compte sur Github
    2. Pour définir un dépôt distant à un projet existant : git remote add origin "url du depot distant"
    3. Vérifier qu'un projet à un depôt distant git remote
    4. Récuperer un projet sur Github git pull origin master
